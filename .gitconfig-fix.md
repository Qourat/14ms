# Fix Git Editor Issue in Cursor

## Problem
Cursor is trying to use a shell script (`git-editor.sh`) that doesn't work on Windows.

## Solution 1: Override in Cursor Settings

1. Open Cursor Settings:
   - Press `Ctrl+,` (or File → Preferences → Settings)
   
2. Search for: `git.useEditorAsCommitInput`
   
3. **Uncheck** or set to `false`:
   - `Git: Use Editor As Commit Input` = `false`

4. Search for: `git.editor`
   
5. Set to: `notepad.exe` or leave empty

## Solution 2: Use Git Command Line with -m flag

Instead of letting Git open an editor, use the `-m` flag:

```bash
git commit -m "Your commit message here"
```

## Solution 3: Set Git Config (Already Done)

The following has been set:
```bash
git config --global core.editor "notepad.exe"
```

## Solution 4: Use VS Code as Editor (if installed)

```bash
git config --global core.editor "code --wait"
```

## Solution 5: Create a Batch File Wrapper

Create `C:\Users\POWER.KEY\git-editor.bat`:
```batch
@echo off
notepad.exe %1
```

Then set:
```bash
git config --global core.editor "C:/Users/POWER.KEY/git-editor.bat"
```

## Verify Current Settings

Run:
```bash
git config --global --get core.editor
```

Should show: `notepad.exe` or `C:\Windows\System32\notepad.exe`
